1. System Overview & Mission

Campus AI is an enterprise-grade academic chatbot built specifically for university environments, student support departments, and technical engineering curricula. Designed from the ground up by Lead Architect Anas, the platform bridges the gap between ultra-fast rule-based FAQ resolvers and complex generative AI models.

Standard LLM chatbots often suffer from latency (2–5 seconds per response) and high token costs when resolving repetitive campus administrative queries (e.g., tuition deadlines, hostel rules, fee structures). Campus AI solves this by implementing a dual-tier processing engine: an instantaneous local NLP vector classifier for immediate campus FAQs (`~100ms` latency), backed by a state-of-the-art large language model (`Llama-3.3-70B-Versatile` via Groq) for deep academic tutoring in mathematics, algorithms, Python, C++, and complex reasoning.

⚡ Tier 1: Local NLP Engine

Resolves 80%+ of repetitive institutional queries locally in under 100 milliseconds without consuming external cloud API tokens.

🧠 Tier 2: Generative LLM Core

Dynamically handles programming debugging, algorithmic breakdowns, and unstructured academic inquiries with full Markdown rendering.

2. Hybrid System Architecture

When a student submits a query via the frontend UI, the request undergoes multi-stage routing inside `chatbot.py`. Below is the complete step-by-step algorithmic pipeline engineered by Anas:

01

Input Preprocessing & Normalization

The incoming text string is stripped of excessive whitespace, converted to lowercase, and normalized using NLTK tokenization (`nltk.word_tokenize`) to remove noise and punctuation artifacts.

02

TF-IDF Vectorization & Cosine Similarity Evaluation

The cleaned query is transformed into a numerical vector via `scikit-learn`'s `TfidfVectorizer`. The engine computes cosine similarity against all canonical training patterns indexed inside `intents.json`.

03

Threshold Verification (`Confidence >= 0.25`)

If the highest cosine similarity match meets or exceeds the confidence threshold (`0.25`), the system immediately retrieves a randomized verified institutional response from the matched intent tag and returns it to the client.

04

Intelligent LLM Fallback Routing (`Groq API`)

If the query score is below threshold (`< 0.25`) or contains complex technical programming syntax, the request is seamlessly handed off to `Llama-3.3-70B-Versatile` along with an academic system prompt to generate a comprehensive, structured response.

3. Setup & Installation Guide

Campus AI is engineered to run locally on Windows, macOS, or Linux, as well as deploy effortlessly to serverless cloud providers such as Vercel. Follow these steps to initialize the environment:

Local Development Setup

# 1. Clone the repository from GitHub
git clone https://github.com/mdanasnurani/campus-ai-chatbot.git
cd campus-ai-chatbot

# 2. Create and activate a Python virtual environment
python3 -m venv .venv
source .venv/bin/activate  # On Windows: .venv\Scripts\activate

# 3. Install required backend packages
pip install -r requirements.txt

# 4. Launch the local Flask server
python app.py

Once started, open your web browser and navigate to http://localhost:5000 to view the landing page, or http://localhost:5000/app to launch the ChatGPT-inspired interface.

4. NLP Intent Engine & Knowledge Base (`intents.json`)

The local classification engine relies on `intents.json`, which serves as the canonical institutional knowledge repository. Administrators and curriculum developers can extend the bot's capabilities simply by appending new JSON tag blocks without altering Python logic.

Schema Definition

{
  "intents": [
    {
      "tag": "college_fees",
      "patterns": [
        "what is the tuition fee for btech semester",
        "how much does hostel accommodation cost",
        "tell me about fee payment deadlines"
      ],
      "responses": [
        "Tuition fees for the undergraduate B.Tech program are due by August 15th for the Fall semester. Detailed breakdowns can be accessed via the student portal under Accounts."
      ]
    }
  ]
}

5. Groq LLM Fallback & Academic Tutoring

For inquiries outside the scope of static campus administrative rules—such as explaining Object-Oriented Programming (OOP) in Python, solving calculus differential equations, or debugging C++ pointers—Campus AI connects to the Groq inference engine.

The backend utilizes the `Llama-3.3-70B-Versatile` model. Users can dynamically inject their own Groq API keys via the Settings modal (`⚙️`) inside the chat sidebar, allowing zero-downtime key rotation and custom rate-limit management.

6. Frontend Design & UI Specifications

The user interface (`templates/chat.html` + `static/css/style.css`) is built entirely using vanilla HTML5, CSS3, and JavaScript, eliminating heavy framework bundle sizes while achieving a pristine, ChatGPT-accurate dark aesthetic (`#000000` canvas and `#0D0D0D` fixed sidebar).

Specification Value / Implementation Details
Color Palette True Dark Mode (`#000000` main background, `#0D0D0D` sidebar, `#1E293B` message bubbles, `#38BDF8` active accent).
Typography Primary UI text in Inter (`300`–`700` weight); Headings & branding in Outfit (`600`–`800` weight).
Markdown Rendering Integrated marked.min.js dynamically formats headings, bullet lists, bold text, tables, and fenced code blocks (`pre code`).
Responsive Layout Universal desktop & mobile collapse engine (`v23.0`). Sidebar glides to `margin-left: -260px` on desktop with smooth cubic-bezier transitions.

7. Multi-Session LocalStorage Engine (`v20.0`)

To maintain absolute user privacy and zero server database overhead, Campus AI stores all conversation histories locally within the browser's `localStorage` engine under versioned keys (`campus_ai_sessions_v20` and `campus_ai_msgs_[sessionId]`).

  • Zero Default History: The application initializes with an empty state (`0 items` in Recents), ensuring complete privacy upon fresh launch.
  • Discrete Session Deletion: Each saved conversation row features an integrated trash icon (`🗑️`), allowing selective one-by-one session pruning without bulk data destruction (`v21.0`).
  • Instant Session Switching: Clicking any recent session dynamically swaps the active DOM message tree in under 10 milliseconds.

8. Universal Share Conversation Engine (`v22.0`)

Campus AI includes a comprehensive sharing subsystem engineered by Anas, allowing students to collaborate and distribute AI tutoring insights across multiple platforms:

  • Native Mobile / Desktop OS Sharing: Clicking the top-right `↗ Share` button invokes the browser's native `navigator.share` API, enabling direct transmission via WhatsApp, Telegram, Mail, or iMessage.
  • Automatic Clipboard Fallback: On desktop browsers lacking native OS share dialogs, the system automatically compiles a clean Markdown summary of the entire session and copies both the transcript and live cloud URL directly to the clipboard with a sleek toast notification (`↗️ Conversation summary copied to clipboard!`).
  • Per-Message Sharing: Every bot response features its own independent share icon alongside `Copy`, `Like`, and `Regenerate` controls.

9. Backend API Reference (`/chat`)

The primary endpoint accepts POST requests containing the student's prompt and returns structured JSON containing the generated answer, classified intent tag, and confidence score.

HTTP POST `/chat`

Request Headers: Content-Type: application/json

// Request Body Payload
{
  "message": "Explain recursion in Python with a clean example."
}
// Response Body Payload (200 OK)
{
  "response": "Recursion occurs when a function calls itself to solve smaller subproblems...",
  "intent": "groq_llm_fallback",
  "confidence": 0.89,
  "timestamp": "01:25 AM"
}
